gitee基础操作

配置SSH公钥

使用git进行项目版本控制常用github或gitee，gitee不用翻墙，从零开始的第一步是配置SSH公钥。具体步骤如下：
1.在gitee上注册账户
2.在本地创建项目文件夹，在该文件夹下右击空白处，点击"Git Bash Here"
3.在git bash交互界面生成SSHkey，命令如下：

ssh-keygen -t rsa -C "xxxxx@xxxxx.com"   #输入个人邮箱


4.生成后输入以下命令展示公钥，并自行复制

cat ~/.ssh/id_rsa.pub

5.回到gitee，点击头像->设置->SSH公钥，粘贴公钥并确定。完成公钥配置。

同步仓库到本地

完成公钥配置之后，我们需要将本地与仓库建立连接，并将仓库内容同步到本地。具体步骤如下：
1.设置用户名与邮箱

git config --global user.name "您的 Gitee 账户姓名"
git config --global user.email "您注册/绑定的邮箱地址"

2.在gitee自己的仓库中点击克隆/下载，选择SSH并复制
3.初始化本地仓库并克隆远程仓库

git init                                #初始化仓库
git remote add origin 粘贴复制的SSH地址  #建立远程连接
git clone 粘贴复制的SSH地址              #克隆远程仓库

此时在本地项目文件夹中可以看到克隆下来的项目。完成同步仓库到本地。

本地内容更新后同步到仓库

当本地内容更新后，我们需要同时把仓库内容进行更新。具体步骤如下：
1.本地文件内容更新，比如增加了文件clean.py，在该文件夹下打开git bash（打开是master分支）
2.通过以下指令进行数据同步

git add clean.py             #新增“clean.py”至暂存区
git commit -m "新纪录"       #确认新增“clean.py”至数据目录
git push -u origin master   #推送新增文件到远程仓库

过去实践中总结的通用步骤

1.ssh key添加
2.git clone (默认是从master上) # 将远程项目克隆到本地
3.本地创建要上传的项目文件，并通过git bash进入该文件目录
4.git checkout "分支名" # 切换到自己的分支 (用git branch "分支名" 来创建分支)
5.git pull origin "分支名" # 拉取最新分支
6.本地项目内容更新了，需要远端同步，git add . (添加当前工作目录文件到index，注意git add *和它是有区别的) # 添加到暂存区
7.git commit -m "xxx" # 将暂存区内容加到数据目录
8.git push origin "分支名" # 将本地的分支版本上传到远程并合并

git add . 会把当前目录及子孙目录里的变动都加到暂存区；而 git add --all 会将项目里所有文件的变动都加到暂存区，也就是说该命令不论在项目的哪级目录执行，都有同样的效果
git add . 会把本地所有untrack的文件都加入暂存区，并且会根据.gitignore做过滤;git add * 会忽略.gitignore把任何文件都加入.
